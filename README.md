# ENTcapture2

ENTcapture2 は、DirectShow 対応のビデオキャプチャデバイスから、動画や静止画を取り込むための Windows 用キャプチャツールです。

医療機関の現場で、内視鏡・顕微鏡・USBカメラ・キャプチャカードから画像を保存し、RSBase へファイリングする用途を想定しています。
このソフト自体は画像管理データベースを持たず、撮影した静止画・動画ファイルを指定フォルダ、特に RSBase の `gazou` (または`gazouauto`)フォルダへ`12345~0001~2026_06_25~内視鏡~RSB.jpg`のようなRSBaseの自動ファイリングフォーマットで保存する方針です。

ENTcapture2 は [ENTcapture Ver1](https://github.com/YUKI-ENT/ENTcapture1) の後継版として、.NET / WinForms ベースで作り直しているバージョンです。

## 主な特徴

- DirectShow 対応キャプチャデバイスからのプレビュー、静止画保存、動画録画
- RSBase と患者ID・患者名を同期(RSBaseの患者遷移に同期してID患者名自動設定) 
- RSBase への自動ファイリング、取込通知
- 静止画への日付、患者ID、患者名などの文字埋め込み
- ROI 指定による画像の一部切り出し
- ホワイトバランス、ガンマ補正、上下左右反転
- 動画の再生、区間指定、簡易クリップ書き出し
- フットスイッチやキーボード割り当てによる撮影操作

## 注意点

Ver1でもわかりにくかったかもしれませんが、本ソフトのコンセプトは「**記録失敗（再検査）を極力避けること**」です。

なので、プレビュー中も一時録画ファイルを作り、検査中に静止画撮り忘れがあっても後から再生画面から静止画キャプチャや動画切り出しを行えるような設計にしています。

この機能が不要な場合は、手間ではありますが`プレビューのみ`の設定も可能なようにオプションは設定していますが、ご理解いただきますようお願いします。

動画の扱いについては以下の3パターンが選択可能です。
- 一時録画も作らずプレビューのみ
- 一時録画をつくり、後から再生や静止画・動画の切り出しが可能（デフォルト）
- 一時録画をつくり、動画をRSBaseに送信、保存

## ENTcapture2 で強化した点

ENTcapture Ver1 から、特に次の点を強化しています。

### 1. プリセットを上限なく登録可能

Ver1 では用途ごとの設定数に制限がありましたが、ENTcapture2 ではプリセットを自由に追加できます。

<img width="1382" height="181" alt="003" src="https://github.com/user-attachments/assets/dcebf1ee-2085-44d4-a096-d91aac6f7df7" />

たとえば、以下のような設定を用途ごとに分けられます。

- ファイバー内視鏡
- 眼振動画
- 手術顕微鏡
- 外部キャプチャカード
- 静止画のみのプレビュー用設定
- 動画ファイリング用設定
- USBカメラ等

プリセットごとに、デバイス、解像度、録画有無、ROI、文字埋め込み、補正値などを保存できます。

### 2. ffmpeg による動画処理

動画処理エンジンとして ffmpeg を利用します。

環境に応じて、H.264 などの一般的な形式への保存や、NVENCやAMF,QSV等GPU ハードウェア支援を利用したエンコードを選択できます。
低負荷で動画保存したい場合や、長時間録画を扱う場合に有利です。

### 3. ベストショット自動判定

静止画キャプチャ時に、直前の短い時間幅から、ブレが少ないフレームを自動選択する機能を追加しています。

標準では 300ms の範囲から候補フレームを比較します。
ROI が設定されている場合は、主にその範囲を参考にして、よりシャープなフレームを選びます。

設定値を `0` にすると、この機能は無効になります。

### 4. 簡易NBIフィルタ

通常のRGB画像に対して、赤成分を抑え、青・緑成分と局所コントラストを強調する簡易フィルタを追加しています。

本物の NBI は光源側の狭帯域光による観察であり、**通常画像から完全に再現するものではありませんので保険請求も臨床診断にも使えない**ことはご了承ください。
ENTcapture2 の簡易NBIは、あくまで視認性調整用の画像処理フィルタです。

<img width="1382" height="1004" alt="021" src="https://github.com/user-attachments/assets/5abb936d-a3e8-424a-b79f-aa74fd5317f2" />

### 5. 動画編集機能の強化

Ver1では動画クリップの1箇所しか切り出しができませんでしたが、複数個所から切り出しができるようになりました。

また、品質指定とビットレート指定を選べるようにしました。

<img width="906" height="753" alt="image" src="https://github.com/user-attachments/assets/de502913-37de-4dda-a69f-6b87f4a8805b" />

### 6. 可変フレームレート動画に対応

USBカメラ等フレームレートが一定でないソースでも、可変フレームレート動画として録画することにより、正確な速度で再生できる動画を出力できるようにしました。


## 対応環境

### 実行環境

- Windows 10 / Windows 11
- 64bit Windows
- DirectShow 対応のキャプチャデバイス
- メモリ 4GB 以上推奨

インストーラ版は .NET ランタイムを同梱する自己完結型として配布する想定です。
通常利用では、別途 .NET SDK をインストールする必要はありません。

### 開発環境

- .NET 10 SDK
- .NET 10 対応の Visual Studio
- Inno Setup 6
- Windows 10 / 11 x64

## 動作確認を想定している機器

DirectShow 対応機器であれば利用できる可能性があります。

Ver1 では、以下のような機器で利用されていました。

- IEEE1394 DV キャプチャ
- ADVC300
- Twinpact110
- Name card scanner BC-01
- Logicool C270
- USB3HDCAP
- OMAX C-mount microscope camera
- キーボード割り当て可能な USB フットスイッチ

ENTcapture2 でも同様に、Windows から DirectShow デバイスとして認識される機器を対象とします。

## 基本的な使い方

### 1. キャプチャデバイスを接続する

USBカメラ、内視鏡キャプチャ、HDMIキャプチャカードなどを PC に接続します。

Windows のカメラアプリやメーカー付属ソフトで映像が出ることを確認しておくと、トラブルシュートしやすくなります。

### 2. ENTcapture2 を起動する

起動時は、登録済みプリセットの先頭が選択されます。

アプリは多重起動しないようになっています。
すでに起動中の場合は、2つ目のENTcapture2は起動しません。

### 3. プリセットを作成する

設定画面からプリセットを追加し、以下を設定します。

- 使用するキャプチャデバイス
- 解像度
- 動画ファイリングの有無
- プレビューのみの有無
- ROI
- 文字埋め込み
- ホワイトバランス、ガンマ、反転、簡易NBI

動画ファイリングを行う場合は一時録画が必要です。
そのため、`動画ファイリング` と `プレビューのみ` は同時に ON にならないようになっています。

<img width="1242" height="972" alt="image" src="https://github.com/user-attachments/assets/7fe76587-db1f-4b56-bf21-5b846cf0ded2" />

<img width="1242" height="972" alt="202" src="https://github.com/user-attachments/assets/2d77d66e-6e05-4918-94cd-d7b536535f93" />


### 4. プレビューを開始する

メイン画面でプリセットを選び、開始ボタンを押すとプレビューが始まります。

プレビュー中に静止画保存ボタン、または設定済みホットキー・フットスイッチで静止画を保存できます。

### 5. 静止画を保存する

静止画は設定した出力フォルダに JPEG として保存されます。

プリセットでROIや文字埋め込みを設定している場合は、保存時に反映されます。
ベストショット自動判定が有効な場合は、OpenCVの機能を使って直前フレームの中からブレが少ないものを選んで保存します。

<img width="640" height="480" alt="12347~0147~2026_06_28~内視鏡~RSB" src="https://github.com/user-attachments/assets/d46826b6-adb0-49aa-9fe3-7dccdc854a3c" />

※ `プレビューのみ` が設定されている場合はベストショット機能は利用できません

### 6. 動画を保存する

動画ファイリングを有効にしたプリセットでは、プレビュー中に一時録画を行い、終了時に動画ファイルとして保存・ファイリングします。

ffmpeg を利用できる環境では、H.264 などの形式やハードウェア支援エンコードを利用できます。

## RSBase 連携

ENTcapture2 は RSBase との連携を前提にしています。

主な連携内容は以下です。

- RSBase 側の患者ID、患者名の取得
- 保存ファイル名への患者情報反映
- RSBase の `gazou` フォルダへの保存
- 取込URLの呼び出し
- 必要に応じて、標準ブラウザで患者ページを開く

患者ページURLは、患者IDが `1234` の場合、以下のような形式です。

```text
http://127.0.0.1/~rsn/N2017.cgi?id_change=1234==
```

出力フォルダに画像・動画ファイルがない場合は、RSBase自動ファイリングや患者フォルダ表示を行わないようにしています。

## プレビューのみモード

`プレビューのみ` は、一時録画ファイルを作らず、画面表示と静止画キャプチャだけを行うモードです。

ENTcapture2 では、この設定は全体設定ではなくプリセットごとに持ちます。

- プレビューだけしたい機器
- 静止画だけ保存したい検査
- 動画保存が不要な用途

このような場合に使います。

## ROI と文字埋め込み

プリセットごとに ROI と文字埋め込みを設定できます。

ROI は、画像の一部だけを切り出して保存する機能です。
設定画面ではサンプル画像を読み込み、画像上をクリックして座標を確認できます。

文字埋め込みでは、日付、患者ID、患者名などを画像上に書き込めます。

## 補正・フィルタ

メインフォーム右端の `<補正` サイドバーでは、以下を調整できます。

- R / G / B のバランス
- ガンマ補正
- 左右反転
- 上下反転
- 簡易NBI

  <img width="822" height="564" alt="image" src="https://github.com/user-attachments/assets/da037cd8-24b2-4a98-b769-497ba693e54f" />

ホワイトバランスは、白を設定したいエリアをマウスボタンでドラッグ選択すると、その部分が白になるように補正します。

設定した補正値はプリセットに保存できます。
検査機器ごとに色味が違う場合は、プリセットを分けておくと便利です。

## 拡大表示フォーム

プレビュー画面をダブルクリックすると、拡大表示フォームを開けます。

- マウスホイールで拡大・縮小
- ウィンドウサイズ変更に合わせて画像をフィット
- 大きく拡大した場合はスクロールバーで表示範囲を移動、＜＞ボタンや矢印キーでコマ送りが可能
- OpenCV の **Lanczos リサイズで高画質表示** しています

<img width="1218" height="986" alt="011" src="https://github.com/user-attachments/assets/83a2a80f-ff0f-41b3-81cc-d511c28bdafe" />


## 設定ファイルとログ

ENTcapture2 の設定やログは、ユーザーごとの AppData 配下に保存されます。

```text
%LOCALAPPDATA%\YUKI_ENT_clinic\ENTcapture2
```

主な保存内容は以下です。

- 設定JSON
- 患者情報キャッシュ用SQLite DB
- ログファイル

ログは、デバイス接続失敗、録画エラー、ベストショット判定などの確認に使います。

## Ver1 からの移行

ENTcapture2 には、旧ENTcaptureの設定インポート機能があります。

<img width="527" height="233" alt="image" src="https://github.com/user-attachments/assets/f36aaf4f-82a0-4b31-a708-228957b4ec90" />


移行時は、Ver1 側で設定をエクスポートしてから、ENTcapture2 の設定画面で読み込んでください。

移行後は、以下を確認してください。

- 出力フォルダ
- RSBase の gazou フォルダ
- RSBase 取込URL
- 使用デバイス
- 解像度
- プリセットごとの動画ファイリング設定
- ROI と文字埋め込み位置

## インストール

配布版では、インストーラを実行してインストールします。

バージョンアップ時は、念のため設定をエクスポートしてから更新してください。

推奨手順:

1. 旧バージョンの設定画面で設定をエクスポートする
2. 必要に応じて旧バージョンをアンインストールする
3. 新しいインストーラを実行する
4. ENTcapture2 を起動する
5. 設定画面から必要に応じて設定をインポートする
6. プリセットと保存先を確認する

## ビルド方法

開発者向けの手順です。

```powershell
dotnet build .\ENTcapture2.sln
```

WinFormsプロジェクトのみをビルドする場合:

```powershell
dotnet build .\src\ENTcapture2.WinForms\ENTcapture2.WinForms.csproj
```

アプリ実行中に exe がロックされる場合は、以下のように `UseAppHost=false` を指定すると確認しやすくなります。

```powershell
dotnet build .\src\ENTcapture2.WinForms\ENTcapture2.WinForms.csproj /p:UseAppHost=false
```

## インストーラ作成

Inno Setup 6 を使用します。

```powershell
.\build\build_installer.ps1
```

署名を省略して確認する場合:

```powershell
.\build\build_installer.ps1 -SkipSign
```

## 注意事項

- 本ソフトは診療補助のためのキャプチャツールです。
- 画像診断そのものを自動化するものではありません。
- 簡易NBIは通常画像に対する画像処理フィルタであり、狭帯域光観察そのものではありません。
- 保存先、患者ID、患者名、RSBase取込結果は運用前に必ず確認してください。
